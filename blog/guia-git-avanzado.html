Introducción

Git es mucho más que un simple sistema de control de versiones. Para desarrolladores profesionales, dominar Git avanzado significa la diferencia entre un caos gestionable y un desarrollo eficiente. Esta guía cubre técnicas avanzadas que transformarán tu forma de trabajar con código.

ℹ️ Prerrequisitos
Esta guía asume que ya conoces los conceptos básicos de Git: commit, push, pull, branch, merge. Si no, revisa primero nuestra guía básica de Git.

Configuración Avanzada

Configuración Global vs Local

Configuración multinivel
# Configuración global (afecta a todos tus repos)
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

# Configuración local (solo este repositorio)
git config user.name "Nombre Profesional"
git config user.email "profesional@empresa.com"

# Configuración a nivel de sistema
git config --system core.editor "vim"

# Ver todas las configuraciones
git config --list --show-origin

Aliases Personalizados

Aliases para productividad
# Alias comunes
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.unstage 'reset HEAD --'
git config --global alias.last 'log -1 HEAD'
git config --global alias.visual '!gitk'

# Alias avanzados
git config --global alias.graph 'log --oneline --graph --decorate --all'
git config --global alias.amend 'commit --amend --no-edit'
git config --global alias.force-push 'push --force-with-lease'
git config --global alias.undo 'reset --soft HEAD~1'

Configuración de Editor y Herramientas

Configuración de herramientas
# Configurar editor por defecto
git config --global core.editor "code --wait"  # VS Code
git config --global core.editor "vim"          # Vim
git config --global core.editor "nano"         # Nano

# Configurar herramienta de diff
git config --global diff.tool "vscode"
git config --global difftool.vscode.cmd 'code --wait --diff $LOCAL $REMOTE'

# Configurar herramienta de merge
git config --global merge.tool "vscode"
git config --global mergetool.vscode.cmd 'code --wait $MERGE'

# Configurar manejo de líneas finales
git config --global core.autocrlf input  # Mac/Linux
git config --global core.autocrlf true   # Windows

# Ignorar permisos de archivos
git config --global core.filemode false

Branching Estratégico

Git Flow vs GitHub Flow

🌿 Git Flow (Vincent Driessen)
  • main/master: Producción estable
  • develop: Integración continua
  • feature/*: Nuevas funcionalidades
  • release/*: Preparación de lanzamiento
  • hotfix/*: Arreglos urgentes
🐙 GitHub Flow
  • main: Siempre desplegable
  • feature branches: Desarrollo
  • Pull Requests: Revisión y merge
  • Deploy: Directo desde main

Creación de Branches Estratégicos

Workflows de branching
# Crear feature branch desde develop
git checkout develop
git pull origin develop
git checkout -b feature/nueva-funcionalidad

# Crear release branch
git checkout develop
git checkout -b release/v1.2.0

# Crear hotfix desde main
git checkout main
git pull origin main
git checkout -b hotfix/critico-bug

# Mover commits entre branches
git checkout target-branch
git cherry-pick 

# Renombrar branch actual
git branch -m nuevo-nombre

# Renombrar branch remoto
git branch -m viejo-nombre nuevo-nombre
git push origin :viejo-nombre nuevo-nombre

Rebase vs Merge

Cuándo usar cada estrategia

⚠️ Regla de Oro
NUNCA hacer rebase en branches compartidos (main, develop, release). Solo haz rebase en tus branches locales antes de hacer merge.

Merge Estratégico

Tipos de merge
# Fast-forward merge (solo si es posible)
git merge feature-branch --ff-only

# No-fast-forward (siempre crea commit de merge)
git merge feature-branch --no-ff

# Squash merge (combina todos los commits en uno)
git merge feature-branch --squash

# Merge con mensaje personalizado
git merge feature-branch -m "Merge: Integrar nueva funcionalidad"

# Abortar merge en conflicto
git merge --abort

Rebase Interactivo

Rebase interactivo avanzado
# Rebase interactivo de los últimos 5 commits
git rebase -i HEAD~5

# Comandos disponibles en rebase interactivo:
# p, pick = usar commit
# r, reword = usar commit pero editar mensaje
# e, edit = usar commit pero permitir modificarlo
# s, squash = usar commit pero combinarlo con el anterior
# f, fixup = como squash pero descarta mensaje
# d, drop = eliminar commit

# Ejemplo de rebase interactivo:
pick 1a2b3c4 Commit inicial
r 5d6e7f8 Corregir typo
s 9a0b1c2 Agregar tests
f 3d4e5f6 Actualizar docs

# Rebase de branch local sobre main
git checkout feature-branch
git fetch origin
git rebase origin/main

# Continuar rebase después de resolver conflictos
git rebase --continue

# Abortar rebase
git rebase --abort

Stashing Avanzado

Gestión Múltiple de Stashes

Stashing con contexto
# Stash con mensaje descriptivo
git stash push -m "Trabajo en progreso: autenticación"

# Stash incluyendo archivos no trackeados
git stash push -u -m "Incluir nuevos archivos"

# Stash solo archivos específicos
git stash push -m "Configuración temporal" -- config/

# Stash con staged y unstaged por separado
git stash push --staged -m "Cambios staged"
git stash push -m "Cambios unstaged"

# Listar todos los stashes
git stash list

# Aplicar stash específico
git stash apply stash@{2}

# Aplicar y eliminar stash
git stash pop stash@{1}

# Ver contenido de stash sin aplicarlo
git stash show -p stash@{0}

# Crear branch desde stash
git stash branch feature-from-stash stash@{1}

# Eliminar stash específico
git stash drop stash@{2}

# Limpiar todos los stashes
git stash clear

Cherry Pick

Selección Selectiva de Commits

Cherry pick avanzado
# Cherry pick de un commit específico
git cherry-pick 

# Cherry pick sin crear commit
git cherry-pick --no-commit 

# Cherry pick manteniendo autor original
git cherry-pick --keep-author 

# Cherry pick de múltiples commits
git cherry-pick ..

# Cherry pick con edición
git cherry-pick --edit 

# Cherry pick de otro repositorio
git remote add upstream https://github.com/original/repo.git
git fetch upstream
git cherry-pick upstream/main~5..upstream/main

# Abortar cherry pick en conflicto
git cherry-pick --abort

# Continuar cherry pick después de resolver conflictos
git cherry-pick --continue

# Cherry pick revert
git cherry-pick --mainline 1

Git Bisect

Búsqueda Binaria de Bugs

Git bisect workflow
# Iniciar bisect
git bisect start

# Marcar commit actual como buggy
git bisect bad

# Marcar commit conocido como bueno
git bisect good v1.0.0

# Git te moverá al punto medio
# Prueba el código y marca como good/bad
git bisect good  # si funciona
git bisect bad   # si tiene el bug

# Automatizar bisect con script
git bisect run ./test-script.sh

# Ver log del bisect
git bisect log

# Resetear bisect
git bisect reset

# Bisect con rango específico
git bisect start v1.0.0 HEAD
git bisect bad HEAD
git bisect good v1.0.0

# Bisect visual
git bisect visualize
💡 Script de prueba para bisect
test-bug.sh
#!/bin/bash
# Script para detectar bug específico

make build || exit 1
./run-tests --specific-test || exit 1

# Si pasa todas las pruebas, es bueno
exit 0

Git Hooks

Automatización con Hooks

Hooks comunes
# Pre-commit hook
#!/bin/sh
# .git/hooks/pre-commit

# Ejecutar linting
npm run lint || exit 1

# Ejecutar tests
npm test || exit 1

# Verificar archivos grandes
files=$(git diff --cached --name-only | grep '\.js$')
for file in $files; do
    if [ $(stat -c%s "$file") -gt 100000 ]; then
        echo "Error: $file es demasiado grande (>100KB)"
        exit 1
    fi
done

exit 0
Pre-push hook
#!/bin/sh
# .git/hooks/pre-push

# Evitar push a main directamente
protected_branch="main"
current_branch=$(git symbolic-ref HEAD | sed -e 's,.*/\(.*\),\1,')

if [ "$current_branch" = "$protected_branch" ]; then
    echo "Error: No puedes hacer push directamente a $protected_branch"
    echo "Usa pull request en su lugar"
    exit 1
fi

# Ejecutar tests completos
npm run test:full || exit 1

exit 0
Commit-msg hook
#!/bin/sh
# .git/hooks/commit-msg

commit_regex='^(feat|fix|docs|style|refactor|test|chore)(\(.+\))?: .{1,50}'
if ! grep -qE "$commit_regex" "$1"; then
    echo "Invalid commit message format."
    echo "Format: type(scope): description"
    echo "Types: feat, fix, docs, style, refactor, test, chore"
    exit 1
fi

exit 0

Submodules

Gestión de Dependencias Externas

Comandos de submodules
# Agregar submodule
git submodule add https://github.com/user/repo.git lib/repo

# Clonar repo con submodules
git clone --recurse-submodules https://github.com/user/main-repo.git

# Inicializar y actualizar submodules
git submodule init
git submodule update

# Atajo para inicializar y actualizar
git submodule update --init --recursive

# Actualizar submodule a latest commit
cd lib/repo
git pull origin main
cd ../..
git add lib/repo
git commit -m "Update submodule to latest"

# Clonar submodule específico
git submodule update --init --recursive lib/repo

# Ver status de submodules
git submodule status

# Eliminar submodule
git submodule deinit lib/repo
git rm lib/repo
git commit -m "Remove submodule"

# Trabajar con submodules
cd lib/repo
git checkout main
git pull
# hacer cambios...
git commit -m "Update submodule"
cd ../..
git add lib/repo
git commit -m "Update submodule content"

Workflows Profesionales

Workflow de Feature Completa

Workflow completo
# 1. Actualizar develop y crear feature
git checkout develop
git pull origin develop
git checkout -b feature/nueva-funcionalidad

# 2. Desarrollo iterativo
git add .
git commit -m "feat: implementar base de la funcionalidad"

# 3. Stash si necesario cambios temporales
git stash push -m "Configuración local temporal"

# 4. Antes de PR: limpiar historial
git fetch origin develop
git rebase -i origin/develop

# 5. Crear pull request
git push origin feature/nueva-funcionalidad

# 6. Después de aprobación: merge y cleanup
git checkout develop
git pull origin develop
git branch -d feature/nueva-funcionalidad
git push origin --delete feature/nueva-funcionalidad

Workflow de Hotfix

Hotfix urgente
# 1. Crear hotfix desde main
git checkout main
git pull origin main
git checkout -b hotfix/critico-bug

# 2. Arreglar bug rápidamente
git add .
git commit -m "fix: corrección crítica de seguridad"

# 3. Merge a main y develop
git checkout main
git merge --no-ff hotfix/critico-bug
git tag -a v1.2.1 -m "Hotfix: corrección crítica"

git checkout develop
git merge --no-ff hotfix/critico-bug

# 4. Deploy inmediato
git push origin main --tags
git push origin develop

# 5. Cleanup
git branch -d hotfix/critico-bug

Workflow de Release

Preparación de release
# 1. Crear release branch
git checkout develop
git pull origin develop
git checkout -b release/v2.0.0

# 2. Actualizar versión y changelog
echo "2.0.0" > VERSION
# actualizar CHANGELOG.md
git add VERSION CHANGELOG.md
git commit -m "chore: preparar versión 2.0.0"

# 3. Testing y bugfixes
# ... ciclos de test y fix ...

# 4. Merge a main y tag
git checkout main
git merge --no-ff release/v2.0.0
git tag -a v2.0.0 -m "Release versión 2.0.0"

# 5. Merge de vuelta a develop
git checkout develop
git merge --no-ff release/v2.0.0

# 6. Deploy y cleanup
git push origin main --tags
git push origin develop
git branch -d release/v2.0.0

Conclusiones

Dominar Git avanzado te permite trabajar con mayor eficiencia y confianza. Estas técnicas no solo mejoran tu productividad, sino que también establecen bases sólidas para trabajar en equipos profesionales.

⭐ Puntos Clave

  • Configura Git adecuadamente para cada entorno
  • Usa branching estratégico según el proyecto
  • Distingue cuándo usar merge vs rebase
  • Automatiza con hooks para mantener calidad
  • Usa workflows profesionales para consistencia
🎓 Próximos Pasos
Practica estas técnicas en proyectos personales antes de aplicarlas en equipos. La consistencia y la comunicación son clave para el éxito con Git avanzado.
