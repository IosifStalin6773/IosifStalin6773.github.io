Introducción

Los servidores de gaming son objetivos frecuentes de ataques maliciosos. Desde DDoS hasta exploits de scripts, las amenazas son diversas y constantes. En este artículo exploraremos las mejores prácticas para proteger tus servidores y mantener una experiencia de juego segura para todos los usuarios.

⚠️ Advertencia
La seguridad no es opcional en servidores gaming. Un solo ataque puede comprometer datos de usuarios, causar pérdida de jugadores y dañar tu reputación.

Tipos Comunes de Ataques

Antes de proteger tu servidor, es crucial entender contra qué te estás defendiendo:

💣 1. Ataques DDoS (Distributed Denial of Service)
Qué es: Un atacante utiliza múltiples sistemas comprometidos (botnet) para enviar tráfico masivo a tu servidor.
Consecuencias: El servidor se sobrecarga, causa lag extremo o se cae por completo. Es el ataque más común en servidores gaming.
Ejemplo real: Un servidor de Minecraft recibe 10,000 peticiones por segundo desde miles de IPs diferentes.
💻 2. Exploits de Scripts
Qué es: Los atacantes aprovechan vulnerabilidades en el código del servidor, addons o plugins mal programados.
Consecuencias: Pueden ejecutar comandos no autorizados, duplicar items, expulsar jugadores o incluso tomar control del servidor.
Ejemplo real: Un plugin de economía mal validado permite a un jugador ejecutar ;give 1000 gold sin permisos.
🗄️ 3. Inyección SQL
Qué es: El atacante inserta código SQL malicioso en campos de entrada (login, chat, formularios).
Consecuencias: Puede leer, modificar o eliminar toda tu base de datos, robando credenciales de usuarios.
Ejemplo real: En un login, el usuario ingresa ' OR '1'='1 como contraseña, eludiendo la verificación.
🖥️ 4. RCE (Remote Code Execution)
Qué es: El atacante logra ejecutar código arbitrario en el sistema operativo del servidor.
Consecuencias: Es el ataque más peligroso posible. El atacante puede instalar malware, robar archivos, o usar tu servidor para atacar a otros.
Ejemplo real: Un comando de servidor mal sanitizado permite ejecutar ; rm -rf / (eliminar todo el sistema).
🔑 5. Brute Force (Fuerza Bruta)
Qué es: El atacante intenta miles o millones de combinaciones de contraseñas por segundo.
Consecuencias: Pueden comprometer cuentas administrativas (RCON, FTP, SSH) o cuentas de jugadores VIP.
Ejemplo real: Un bot prueba 10,000 contraseñas por minuto contra el puerto SSH de tu servidor.

Configuración Básica de Seguridad

1. Firewall y Puertos 🐧 Linux - iptables

¿Qué es un firewall? Es un sistema que filtra el tráfico de red, permitiendo o bloqueando conexiones según reglas definidas. Regla de oro: Solo abre los puertos estrictamente necesarios.

Configuración de Firewall (iptables) - Linux
# 🔒 Permitir solo puertos esenciales
iptables -A INPUT -p tcp --dport 22 -j ACCEPT      # SSH (administración remota)
iptables -A INPUT -p tcp --dport 27015 -j ACCEPT   # Puerto del servidor de juego (TCP)
iptables -A INPUT -p udp --dport 27015 -j ACCEPT   # Puerto del servidor de juego (UDP)

# 🔒 Limitar conexiones por IP (anti-DDoS básico)
iptables -A INPUT -p tcp --dport 27015 -m connlimit --connlimit-above 10 -j REJECT

# 🔒 Protección contra ping floods
iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 1/s -j ACCEPT
iptables -A INPUT -p icmp --icmp-type echo-request -j DROP

# 🔒 Bloquear todo lo demás
iptables -P INPUT DROP

Firewall en Windows Server 🪟 Windows - PowerShell

PowerShell - Reglas de Firewall para Windows
# Crear regla para permitir el puerto del juego
New-NetFirewallRule -DisplayName "Game Server TCP" -Direction Inbound -Protocol TCP -LocalPort 27015 -Action Allow
New-NetFirewallRule -DisplayName "Game Server UDP" -Direction Inbound -Protocol UDP -LocalPort 27015 -Action Allow

# Bloquear ICMP (pings)
New-NetFirewallRule -DisplayName "Block ICMP" -Protocol ICMPv4 -IcmpType 8 -Action Block

2. Rate Limiting 📜 Lua - Multiplataforma

¿Qué es rate limiting? Técnica que limita cuántas veces un usuario puede realizar una acción en un período de tiempo.

Lua - Rate limiting para conexiones
local connectionLimiter = {}
local maxConnections = 5        -- Máximo 5 conexiones por IP
local connectionWindow = 60     -- En 60 segundos

function canConnect(ip)
    local now = os.time()
    
    if not connectionLimiter[ip] then
        connectionLimiter[ip] = {count = 1, firstTime = now}
        return true
    end
    
    local data = connectionLimiter[ip]
    
    if now - data.firstTime > connectionWindow then
        data.count = 1
        data.firstTime = now
        return true
    end
    
    if data.count >= maxConnections then
        return false
    end
    
    data.count = data.count + 1
    return true
end

Validación de Entradas

La mayoría de los exploits provienen de entradas no validadas. Implementa validación estricta en todos los puntos de entrada.

🔒 Regla de Oro de Seguridad
Nunca, bajo ninguna circunstancia, confíes en las entradas del usuario. Siempre valida y sanitiza TODOS los datos provenientes de clientes.

Sanitización de Strings 📜 Lua

Lua - Función de sanitización robusta
local function sanitizeInput(input, inputType)
    if not input or type(input) ~= "string" then
        return nil
    end
    
    -- Eliminar caracteres peligrosos: < > " ' & y caracteres de control
    local sanitized = string.gsub(input, "[<>\"'&%c]", "")
    
    -- Limitar longitud según el tipo
    local maxLength = 255
    if inputType == "chat" then
        maxLength = 128
    elseif inputType == "name" then
        maxLength = 32
    elseif inputType == "command" then
        maxLength = 64
    end
    
    if #sanitized > maxLength then
        sanitized = string.sub(sanitized, 1, maxLength)
    end
    
    -- Validación específica por tipo
    if inputType == "number" then
        local num = tonumber(sanitized)
        return num and num >= 0 and num <= 1000000 and num or nil
    elseif inputType == "name" then
        if not string.match(sanitized, "^[%w%s]+$") then
            return nil
        end
    end
    
    return sanitized
end

function handleCommand(player, command, args)
    local sanitizedCmd = sanitizeInput(command, "command")
    if not sanitizedCmd then
        player:ChatPrint("Comando inválido")
        return
    end
    processSafeCommand(player, sanitizedCmd, args)
end

Protección contra DDoS

1. Rate Limiting Avanzado 📜 Lua

Sistema anti-DDoS multicapa
local DDoSProtection = {
    ipConnections = {},
    blacklistedIPs = {},
    maxConnectionsPerIP = 3,
    connectionTimeout = 30,
    blacklistDuration = 3600  -- 1 hora
}

function DDoSProtection:checkConnection(ip)
    local now = os.time()
    
    -- Verificar blacklist
    if self.blacklistedIPs[ip] and self.blacklistedIPs[ip] > now then
        return false, "IP bloqueada temporalmente"
    end
    
    -- Limpiar conexiones expiradas
    self:cleanupExpiredConnections()
    
    -- Verificar límite de conexiones
    local connections = self.ipConnections[ip] or {}
    local activeConnections = 0
    
    for _, conn in pairs(connections) do
        if conn.expires > now then
            activeConnections = activeConnections + 1
        end
    end
    
    if activeConnections >= self.maxConnectionsPerIP then
        self:blacklistIP(ip, "Demasiadas conexiones")
        return false, "Límite de conexiones excedido"
    end
    
    table.insert(connections, {expires = now + self.connectionTimeout})
    self.ipConnections[ip] = connections
    return true, "Conexión permitida"
end

function DDoSProtection:blacklistIP(ip, reason)
    self.blacklistedIPs[ip] = os.time() + self.blacklistDuration
    print(string.format("IP %s bloqueada: %s", ip, reason))
end

2. Cloudflare y CDN 🌐 Multiplataforma

Cloudflare actúa como escudo entre tus jugadores y tu servidor, ocultando tu IP real y mitigando ataques DDoS.

  • Beneficios: Oculta IP real, mitiga DDoS, proporciona SSL gratuito
  • Configuración recomendada: Activar "Proxy" (nube naranja), configurar reglas de firewall, activar "I'm Under Attack Mode" durante ataques
ℹ️ Nota
Cloudflare ofrece protección DDoS gratuita suficiente para servidores gaming pequeños y medianos. Para servidores grandes, considera el plan Pro ($20/mes).

Seguridad de Base de Datos

1. Consultas Parametrizadas 🗄️ SQL

Las consultas parametrizadas previenen la inyección SQL al separar el código SQL de los datos.

SQL Injection - Mal vs Bien
-- ❌ MAL - Vulnerable a SQL Injection
-- Si el jugador ingresa "Robert'; DROP TABLE players; --"
local query = "SELECT * FROM players WHERE name = '" .. playerName .. "'"

-- ✅ BIEN - Consulta parametrizada (segura)
local function getPlayerByName(name)
    local stmt = db:prepare("SELECT * FROM players WHERE name = ?")
    stmt:bind_values(name)  -- El '?' se reemplaza por el valor sanitizado
    local result = stmt:step()
    stmt:finalize()
    return result
end

2. Cifrado de Datos Sensibles 📜 Lua con bcrypt

Cifrado de contraseñas con bcrypt
local bcrypt = require("bcrypt")  -- luarocks install bcrypt

function hashPassword(password)
    return bcrypt.digest(password, 12)  -- 12 rounds
end

function verifyPassword(password, hash)
    return bcrypt.verify(password, hash)
end

function registerUser(username, plainPassword)
    local safeUsername = sanitizeInput(username, "name")
    if not safeUsername then return false end
    
    local passwordHash = hashPassword(plainPassword)
    local stmt = db:prepare("INSERT INTO users (username, password_hash) VALUES (?, ?)")
    stmt:bind_values(safeUsername, passwordHash)
    stmt:step()
    return true
end

Monitoreo y Logs

1. Sistema de Logging 📜 Lua

Sistema de logging seguro
local Logger = {
    logFile = "server_security.log",
    logLevels = {DEBUG=1, INFO=2, WARN=3, ERROR=4, CRITICAL=5}
}

function Logger:log(level, message, ip, playerId)
    if self.logLevels[level] < self.logLevels.INFO then return end
    
    local timestamp = os.date("%Y-%m-%d %H:%M:%S")
    local logEntry = string.format("[%s] %s | IP: %s | Player: %s | %s\n", 
                                   timestamp, level, ip or "unknown", 
                                   playerId or "unknown", message)
    
    local file = io.open(self.logFile, "a")
    if file then file:write(logEntry) file:close() end
    
    if level == "CRITICAL" then
        self:sendAlert(message, ip)
    end
end

2. Detección de Anomalías 📜 Lua

Sistema de detección de anomalías
local AnomalyDetector = {
    thresholds = {
        maxCommandsPerSecond = 10,
        suspiciousPatterns = {"admin", "ban", "kick", "exploit"}
    }
}

function AnomalyDetector:checkCommandFrequency(player, command, now, stats)
    local commandCount = 0
    for _ in pairs(stats.commands) do commandCount = commandCount + 1 end
    
    if commandCount > self.thresholds.maxCommandsPerSecond then
        Logger:log("WARN", "Exceso de comandos: " .. commandCount .. "/s", 
                  player:IPAddress(), player:SteamID())
        
        stats.suspiciousActions = (stats.suspiciousActions or 0) + 1
        if stats.suspiciousActions >= 5 then
            self:handleSuspiciousPlayer(player, "Exceso de comandos")
        end
    end
end

Hardening del Servidor

¿Qué es hardening? El proceso de asegurar un sistema reduciendo su superficie de ataque.

1. Configuración de Sistema Operativo 🐧 Linux - /etc/sysctl.conf

/etc/sysctl.conf - Optimizaciones de seguridad
# Protección contra SYN floods
net.ipv4.tcp_syncookies = 1

# Ignorar pings (previene ping floods)
net.ipv4.icmp_echo_ignore_all = 1

# Limitar conexiones concurrentes
net.ipv4.tcp_max_syn_backlog = 2048

# Protección contra IP spoofing
net.ipv4.conf.all.rp_filter = 1

# Deshabilitar redirects ICMP
net.ipv4.conf.all.accept_redirects = 0
net.ipv4.conf.all.send_redirects = 0

# Aplicar cambios: sudo sysctl -p

Comandos de Emergencia 🎮 GMod/Lua

Script de emergencia
local EmergencyCommands = {}

function EmergencyCommands:lockdown()
    -- Bloquear nuevas conexiones
    game.ConsoleCommand("sv_password lockdown_temp_" .. os.time())
    
    -- Expulsar jugadores no administradores
    for _, ply in ipairs(player.GetAll()) do
        if not ply:IsAdmin() then
            ply:Kick("🔒 BLOQUEO DE EMERGENCIA")
        end
    end
    
    Logger:log("CRITICAL", "Bloqueo de emergencia activado", "SYSTEM")
end

function EmergencyCommands:emergencyBackup()
    local backupName = "emergency_backup_" .. os.date("%Y%m%d_%H%M%S")
    os.execute("mysqldump -u root -p gaming_db > backups/" .. backupName .. ".sql")
    os.execute("tar -czf backups/" .. backupName .. "_config.tar.gz server.cfg addons/")
    Logger:log("INFO", "Backup creado: " .. backupName, "SYSTEM")
end

Mejores Prácticas Resumidas

⭐ Checklist de Seguridad

🔧 Configuración (hacer una vez, revisar mensualmente)

  • ✅ Mantener software actualizado (apt update && apt upgrade o Windows Update)
  • ✅ Usar contraseñas fuertes y únicas (12+ caracteres, mayúsculas, números, símbolos)
  • ✅ Configurar firewall correctamente
  • ✅ Limitar acceso administrativo (solo IPs confiables)

📊 Monitoreo (hacer diariamente)

  • ✅ Implementar logging exhaustivo
  • ✅ Configurar alertas automáticas (Discord webhooks, email)
  • ✅ Monitorear recursos del sistema (htop, Task Manager)
  • ✅ Analizar logs regularmente (grep ERROR security.log)

🛡️ Prevención (implementar gradualmente)

  • ✅ Validar TODAS las entradas de usuario 🔴 ALTA
  • ✅ Usar consultas parametrizadas en BD 🔴 ALTA
  • ✅ Implementar rate limiting 🟡 MEDIA
  • ✅ Realizar backups automáticos diarios 🔴 ALTA
