Seguridad en Servidores Gaming

👤 Oscar de la Cruz
📅 10 Mar 2026
⏱️ 12 min lectura

Introducción

Los servidores de gaming son objetivos frecuentes de ataques maliciosos. Desde DDoS hasta exploits de scripts, las amenazas son diversas y constantes. En este artículo exploraremos las mejores prácticas para proteger tus servidores y mantener una experiencia de juego segura para todos los usuarios.

⚠️ Advertencia
La seguridad no es opcional en servidores gaming. Un solo ataque puede comprometer datos de usuarios, causar pérdida de jugadores y dañar tu reputación.

Tipos Comunes de Ataques

Antes de proteger tu servidor, es crucial entender contra qué te estás defendiendo:

💣 1. Ataques DDoS (Distributed Denial of Service)
Sobrecarga del servidor con tráfico masivo para causar lag o caídas. Es el ataque más común en servidores gaming.
💻 2. Exploits de Scripts
Aprovechamiento de vulnerabilidades en el código del servidor o plugins para ejecutar comandos no autorizados.
🗄️ 3. Inyección SQL
Manipulación de bases de datos a través de entradas no validadas, común en servidores con sistemas de cuentas.
🖥️ 4. RCE (Remote Code Execution)
Ejecución remota de código en el servidor, el ataque más peligroso que puede dar control total al atacante.
🔑 5. Brute Force
Intentos repetidos de acceso mediante fuerza bruta a cuentas administrativas o de usuarios.

Configuración Básica de Seguridad

1. Firewall y Puertos

Configuración de Firewall (iptables)
# Permitir solo puertos esenciales
iptables -A INPUT -p tcp --dport 22 -j ACCEPT  # SSH
iptables -A INPUT -p tcp --dport 27015 -j ACCEPT  # Game Server
iptables -A INPUT -p udp --dport 27015 -j ACCEPT  # Game Server UDP

# Limitar conexiones por IP
iptables -A INPUT -p tcp --dport 27015 -m connlimit --connlimit-above 10 -j REJECT

# Protección contra ping floods
iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 1/s -j ACCEPT
iptables -A INPUT -p icmp --icmp-type echo-request -j DROP

2. Rate Limiting

Lua - Rate limiting para conexiones
local connectionLimiter = {}
local maxConnections = 5
local connectionWindow = 60 -- segundos

function canConnect(ip)
    local now = os.time()
    
    if not connectionLimiter[ip] then
        connectionLimiter[ip] = {count = 1, firstTime = now}
        return true
    end
    
    local data = connectionLimiter[ip]
    
    if now - data.firstTime > connectionWindow then
        data.count = 1
        data.firstTime = now
        return true
    end
    
    if data.count >= maxConnections then
        return false
    end
    
    data.count = data.count + 1
    return true
end

Validación de Entradas

La mayoría de los exploits provienen de entradas no validadas. Implementa validación estricta en todos los puntos de entrada.

Nunca confíes en las entradas del usuario. Siempre valida y sanitiza todos los datos provenientes de clientes.

Sanitización de Strings

Lua - Función de sanitización robusta
local function sanitizeInput(input, inputType)
    if not input or type(input) ~= "string" then
        return nil
    end
    
    -- Eliminar caracteres peligrosos
    local sanitized = string.gsub(input, "[<>\"'&%c]", "")
    
    -- Limitar longitud
    local maxLength = 255
    if inputType == "chat" then
        maxLength = 128
    elseif inputType == "name" then
        maxLength = 32
    elseif inputType == "command" then
        maxLength = 64
    end
    
    if #sanitized > maxLength then
        sanitized = string.sub(sanitized, 1, maxLength)
    end
    
    -- Validación específica por tipo
    if inputType == "number" then
        local num = tonumber(sanitized)
        return num and num >= 0 and num <= 1000000 and num or nil
    elseif inputType == "name" then
        -- Solo permitir caracteres alfanuméricos y espacios
        if not string.match(sanitized, "^[%w%s]+$") then
            return nil
        end
    end
    
    return sanitized
end

-- Uso en comandos
function handleCommand(player, command, args)
    local sanitizedCmd = sanitizeInput(command, "command")
    if not sanitizedCmd then
        player:ChatPrint("Comando inválido")
        return
    end
    
    -- Procesar comando seguro
    processSafeCommand(player, sanitizedCmd, args)
end

Protección contra DDoS

1. Configuración de Rate Limiting Avanzado

Sistema anti-DDoS multicapa
local DDoSProtection = {
    ipConnections = {},
    packetLimits = {},
    blacklistedIPs = {},
    
    -- Configuración
    maxConnectionsPerIP = 3,
    connectionTimeout = 30,
    maxPacketsPerSecond = 100,
    blacklistDuration = 3600
}

function DDoSProtection:checkConnection(ip)
    local now = os.time()
    
    -- Verificar blacklist
    if self.blacklistedIPs[ip] and self.blacklistedIPs[ip] > now then
        return false, "IP blacklisted"
    end
    
    -- Limpiar conexiones expiradas
    self:cleanupExpiredConnections()
    
    -- Verificar límite de conexiones
    local connections = self.ipConnections[ip] or {}
    local activeConnections = 0
    
    for _, conn in pairs(connections) do
        if conn.expires > now then
            activeConnections = activeConnections + 1
        end
    end
    
    if activeConnections >= self.maxConnectionsPerIP then
        self:blacklistIP(ip, "Too many connections")
        return false, "Connection limit exceeded"
    end
    
    -- Registrar nueva conexión
    table.insert(connections, {
        expires = now + self.connectionTimeout,
        id = #connections + 1
    })
    self.ipConnections[ip] = connections
    
    return true, "Connection allowed"
end

function DDoSProtection:blacklistIP(ip, reason)
    self.blacklistedIPs[ip] = os.time() + self.blacklistDuration
    print(string.format("IP %s blacklisted: %s", ip, reason))
end

2. Cloudflare y CDN

Utilizar servicios como Cloudflare puede mitigar ataques DDoS antes de que lleguen a tu servidor:

  • Activar "I'm Under Attack" mode durante ataques
  • Configurar reglas de firewall a nivel de CDN
  • Utilizar rate limiting a nivel de edge
  • Ocultar IP real del servidor
Cloudflare ofrece protección DDoS gratuita que es suficiente para la mayoría de servidores gaming pequeños y medianos.

Seguridad de Base de Datos

1. Consultas Parametrizadas

SQL Injection - Mal vs Bien
-- MAL - Vulnerable a SQL Injection
local query = "SELECT * FROM players WHERE name = '" .. playerName .. "'"

-- BIEN - Consulta parametrizada
local function getPlayerByName(name)
    local stmt = db:prepare("SELECT * FROM players WHERE name = ?")
    stmt:bind_values(name)
    local result = stmt:step()
    stmt:finalize()
    return result
end

2. Cifrado de Datos Sensibles

Cifrado de contraseñas y datos
-- Cifrado de contraseñas
local bcrypt = require("bcrypt")

function hashPassword(password)
    return bcrypt.digest(password, 12) -- 12 rounds
end

function verifyPassword(password, hash)
    return bcrypt.verify(password, hash)
end

-- Cifrado de datos sensibles en la base de datos
local function encryptSensitiveData(data, key)
    -- Implementar cifrado AES-256
    return aes256.encrypt(key, data)
end

Monitoreo y Logs

1. Sistema de Logging

Sistema de logging seguro
local Logger = {
    logFile = "server_security.log",
    maxFileSize = 10 * 1024 * 1024, -- 10MB
    logLevels = {
        DEBUG = 1,
        INFO = 2,
        WARN = 3,
        ERROR = 4,
        CRITICAL = 5
    }
}

function Logger:log(level, message, ip, playerId)
    if self.logLevels[level] < self.logLevels.INFO then
        return -- Solo loggear INFO y superior
    end
    
    local timestamp = os.date("%Y-%m-%d %H:%M:%S")
    local logEntry = string.format("[%s] %s - IP: %s - Player: %s - %s\n", 
                                   timestamp, level, ip or "unknown", 
                                   playerId or "unknown", message)
    
    -- Escribir al archivo
    local file = io.open(self.logFile, "a")
    if file then
        file:write(logEntry)
        file:close()
    end
    
    -- Alertas para eventos críticos
    if level == "CRITICAL" then
        self:sendAlert(message, ip)
    end
end

function Logger:sendAlert(message, ip)
    -- Enviar alerta a administradores
    for _, admin in ipairs(getAdmins()) do
        admin:ChatPrint("[SECURITY ALERT] " .. message .. " (IP: " .. ip .. ")")
    end
    
    -- Opcional: enviar a webhook/email
    self:sendWebhookAlert(message, ip)
end

2. Detección de Anomalías

Sistema de detección de anomalías
local AnomalyDetector = {
    playerStats = {},
    thresholds = {
        maxCommandsPerSecond = 10,
        maxChatMessagesPerMinute = 30,
        maxMovementSpeed = 500,
        suspiciousPatterns = {"admin", "ban", "kick", "exploit"}
    }
}

function AnomalyDetector:analyzePlayerAction(player, actionType, data)
    local steamId = player:SteamID()
    local now = os.time()
    
    if not self.playerStats[steamId] then
        self.playerStats[steamId] = {
            commands = {},
            chatMessages = {},
            lastPosition = player:GetPos(),
            suspiciousActions = 0
        }
    end
    
    local stats = self.playerStats[steamId]
    
    if actionType == "command" then
        self:checkCommandFrequency(player, data, now, stats)
    elseif actionType == "chat" then
        self:checkChatFrequency(player, data, now, stats)
    elseif actionType == "movement" then
        self:checkMovementAnomalies(player, data, stats)
    end
end

function AnomalyDetector:checkCommandFrequency(player, command, now, stats)
    -- Limpiar comandos antiguos (más de 1 segundo)
    local recentCommands = {}
    for timestamp, cmd in pairs(stats.commands) do
        if now - timestamp <= 1 then
            recentCommands[timestamp] = cmd
        end
    end
    stats.commands = recentCommands
    
    -- Contar comandos en el último segundo
    local commandCount = 0
    for _ in pairs(stats.commands) do
        commandCount = commandCount + 1
    end
    
    if commandCount > self.thresholds.maxCommandsPerSecond then
        Logger:log("WARN", "Excessive command usage: " .. commandCount .. "/s", 
                  player:IPAddress(), player:SteamID())
        
        stats.suspiciousActions = stats.suspiciousActions + 1
        
        if stats.suspiciousActions > 5 then
            self:handleSuspiciousPlayer(player, "Excessive command usage")
        end
    end
end

Hardening del Servidor

1. Configuración de Sistema Operativo

/etc/sysctl.conf - Optimizaciones de seguridad
# Protección contra SYN floods
net.ipv4.tcp_syncookies = 1

# Ignorar pings
net.ipv4.icmp_echo_ignore_all = 1

# Limitar conexiones concurrentes
net.ipv4.tcp_max_syn_backlog = 2048

# Protección contra IP spoofing
net.ipv4.conf.all.rp_filter = 1

# Deshabilitar redirects ICMP
net.ipv4.conf.all.accept_redirects = 0
net.ipv4.conf.all.send_redirects = 0

Comandos de Emergencia

Script de emergencia para GMod
local EmergencyCommands = {}

function EmergencyCommands:lockdown()
    -- Bloquear nuevas conexiones
    game.ConsoleCommand("sv_password lockdown_temp_" .. os.time())
    
    -- Kickear todos los jugadores excepto admins
    for _, ply in ipairs(player.GetAll()) do
        if not ply:IsAdmin() then
            ply:Kick("Server lockdown - Try again later")
        end
    end
    
    Logger:log("CRITICAL", "Server lockdown activated", "SYSTEM")
end

function EmergencyCommands:backup()
    -- Crear backup de datos críticos
    local backupName = "emergency_backup_" .. os.date("%Y%m%d_%H%M%S")
    
    -- Backup base de datos
    os.execute("mysqldump -u root -p gaming_db > backups/" .. backupName .. ".sql")
    
    -- Backup archivos de configuración
    os.execute("tar -czf backups/" .. backupName .. "_config.tar.gz server.cfg addons/")
    
    Logger:log("INFO", "Emergency backup created: " .. backupName, "SYSTEM")
end

Mejores Prácticas Resumidas

⭐ Checklist de Seguridad

Configuración

  • Mantener software actualizado
  • Usar contraseñas fuertes y únicas
  • Configurar firewall correctamente
  • Limitar acceso administrativo

Monitoreo

  • Implementar logging exhaustivo
  • Configurar alertas automáticas
  • Monitorear recursos del sistema
  • Analizar logs regularmente

Prevención

  • Validar todas las entradas
  • Usar consultas parametrizadas
  • Implementar rate limiting
  • Realizar backups regulares
🎓 Próximos Pasos
La seguridad es un proceso continuo, no un destino. Implementa estas medidas gradualmente y mantente actualizado sobre nuevas amenazas y técnicas de protección.
