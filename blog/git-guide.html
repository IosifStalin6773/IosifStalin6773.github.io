Guía de Git para Desarrolladores

Git Version Control Workflow DevOps

Git es la herramienta de control de versiones más utilizada en el desarrollo de software moderno. Dominar Git no solo significa conocer los comandos básicos, sino entender flujos de trabajo eficientes y técnicas avanzadas que mejorarán tu productividad y la calidad de tu código.

Fundamentos de Git

Antes de entrar en comandos avanzados, asegurémonos de entender los conceptos fundamentales:

Repository (Repositorio)

Un repositorio Git contiene todos los archivos de tu proyecto junto con el historial completo de cambios. Cada repositorio tiene una carpeta .git que almacena toda la información.

Working Directory, Staging Area, Repository

Git tiene tres áreas principales:

  • Working Directory: Donde modificas tus archivos
  • Staging Area: Área donde preparas cambios para el siguiente commit
  • Repository: Donde Git guarda permanentemente tus commits

Commit

Un commit es una instantánea de tu proyecto en un momento específico. Cada commit tiene un ID único (hash) y contiene los cambios, autor, fecha y mensaje.

Branch (Rama)

Las ramas permiten trabajar en diferentes versiones del proyecto simultáneamente. La rama principal suele llamarse main o master.

Configuración Inicial

Configurar Identidad

# Configurar nombre y email (obligatorio)
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

# Configurar editor por defecto
git config --global core.editor "code --wait"  # VS Code
git config --global core.editor "nano"         # Nano
git config --global core.editor "vim"          # Vim

# Configurar herramienta de diferencias
git config --global diff.tool "vscode"

# Configurar alias útiles
git config --global alias.st "status"
git config --global alias.co "checkout"
git config --global alias.br "branch"
git config --global alias.cm "commit -m"
git config --global alias.unstage "reset HEAD --"
git config --global alias.last "log -1 HEAD"
git config --global alias.visual "!gitk"

# Ver configuración actual
git config --list

Configuración de Seguridad

# Configurar SSH keys para GitHub
ssh-keygen -t ed25519 -C "tu@email.com"
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# Copiar clave pública para GitHub
cat ~/.ssh/id_ed25519.pub

# Configurar GPG para firmar commits
gpg --full-generate-key --default-key-new "rsa4096"
git config --global user.signingkey TU_GPG_KEY_ID
git config --global commit.gpgsign true
git config --global tag.gpgsign true

Comandos Esenciales

Operaciones Básicas

# Inicializar repositorio
git init

# Clonar repositorio
git clone https://github.com/user/repo.git
git clone git@github.com:user/repo.git  # SSH

# Ver estado
git status
git status --short          # Vista compacta
git status --branch         # Mostrar rama actual

# Añadir archivos
git add archivo.txt
git add .                    # Todos los archivos
git add *.js                 # Por patrón
git add -A                   # Todos los cambios (incluyendo eliminados)

# Hacer commit
git commit -m "Mensaje descriptivo"
git commit -am "Mensaje"     # Añadir y commit (solo archivos existentes)
git commit --amend          # Modificar último commit

# Ver historial
git log
git log --oneline           # Compacto
git log --graph --oneline   # Con gráfico
git log --author="Tu Nombre"
git log --since="2024-01-01"
git log --grep="fix"        # Buscar en mensajes

# Ver cambios
git diff                    # Cambios no preparados
git diff --staged          # Cambios preparados
git diff HEAD~1            # Comparar con commit anterior

Trabajo con Ramas

# Crear y cambiar rama
git branch feature-nueva-funcionalidad
git checkout feature-nueva-funcionalidad
git checkout -b feature-nueva-funcionalidad  # Crear y cambiar

# Listar ramas
git branch
git branch -r              # Ramas remotas
git branch -a              # Todas las ramas

# Fusionar ramas
git checkout main
git merge feature-nueva-funcionalidad

# Eliminar ramas
git branch -d feature-nueva-funcionalidad  # Después de fusionar
git branch -D feature-nueva-funcionalidad  # Forzar eliminación

# Renombrar rama
git branch -m viejo-nombre nuevo-nombre

# Comparar ramas
git diff main..feature
git log main..feature
git log --left-right main..feature

Trabajo con Remotos

Gestión de Remotos

# Ver remotos
git remote -v

# Añadir remoto
git remote add origin https://github.com/user/repo.git
git remote add upstream https://github.com/original/repo.git

# Cambiar URL de remoto
git remote set-url origin git@github.com:user/repo.git

# Eliminar remoto
git remote remove origin

# Renombrar remoto
git remote rename origin upstream

# Ver información de remoto
git remote show origin

Sincronización

# Obtener cambios (sin fusionar)
git fetch origin
git fetch --all             # Todos los remotos
git fetch --prune          # Eliminar ramas remotas eliminadas

# Descargar y fusionar cambios
git pull origin main
git pull --rebase origin main

# Subir cambios
git push origin main
git push -u origin feature-branch  # Configurar upstream
git push --all               # Todas las ramas
git push --tags             # Incluir tags

# Forzar push (usar con cuidado)
git push --force-with-lease origin main
git push --force origin main  # Último recurso

Técnicas Avanzadas

Rebase Interactivo

# Rebase interactivo de últimos 3 commits
git rebase -i HEAD~3

# Comandos disponibles en rebase interactivo:
# pick: mantener el commit como está
# reword: cambiar mensaje del commit
# edit: modificar el commit
# squash: combinar con commit anterior
# fixup: combinar con commit anterior (descartando mensaje)
# drop: eliminar el commit

# Ejemplo de rebase interactivo:
pick 1234567 Implementar función A
reword 2345678 Fix bug en función B
squash 3456789 Agregar tests para función B
drop 4567890 Commit temporal

Stashing

# Guardar cambios temporalmente
git stash
git stash save "Trabajo en progreso"

# Listar stashes
git stash list

# Aplicar stash
git stash apply
git stash pop            # Aplicar y eliminar

# Aplicar stash específico
git stash apply stash@{2}

# Crear rama desde stash
git stash branch nueva-rama stash@{1}

# Limpiar stashes
git stash clear
git stash drop stash@{0}

Cherry-pick

# Aplicar commit específico
git cherry-pick 1234567

# Aplicar múltiples commits
git cherry-pick 1234567..2345678

# Aplicar sin crear commit
git cherry-pick --no-commit 1234567

# Editar mensaje al aplicar
git cherry-pick --edit 1234567

Resolución de Conflictos

Identificar Conflictos

# Ver archivos con conflictos
git status

# Marcadores de conflicto:
<<<<<<< HEAD
tu código
=======
código del otro
>>>>>>> branch-name

Herramientas de Resolución

# Ver diferencias en conflicto
git diff --name-only --diff-filter=U

# Usar herramienta de merge
git mergetool

# Marcar conflicto como resuelto
git add archivo-en-conflicto.txt

# Abortar merge/rebase
git merge --abort
git rebase --abort

# Continuar después de resolver conflictos
git merge --continue
git rebase --continue

Estrategias de Resolución

# Aceptar cambios de una rama
git checkout --theirs archivo.txt
git checkout --ours archivo.txt

# Usar versión específica
git show branch:path/to/file > archivo.txt

# Merge de 3 vías manual
git merge-file archivo.txt base.txt their.txt our.txt

Flujos de Trabajo

Git Flow

# Ramas principales:
# main: código de producción
# develop: desarrollo integrado
# feature/*: nuevas funcionalidades
# release/*: preparación de lanzamiento
# hotfix/*: correcciones urgentes

# Iniciar nueva funcionalidad
git checkout develop
git pull origin develop
git checkout -b feature/nueva-funcionalidad

# Finalizar funcionalidad
git checkout develop
git merge --no-ff feature/nueva-funcionalidad
git branch -d feature/nueva-funcionalidad
git push origin develop

# Iniciar release
git checkout develop
git checkout -b release/v1.2.0

# Finalizar release
git checkout main
git merge --no-ff release/v1.2.0
git tag -a v1.2.0 -m "Versión 1.2.0"
git checkout develop
git merge --no-ff release/v1.2.0
git branch -d release/v1.2.0

GitHub Flow

# Flujo simplificado
# 1. Crear rama desde main
git checkout main
git checkout -b feature/nueva-funcionalidad

# 2. Hacer cambios y commits
git add .
git commit -m "Implementar nueva funcionalidad"

# 3. Push y Pull Request
git push origin feature/nueva-funcionalidad
# Crear PR en GitHub

# 4. Revisión y merge
# Después del merge en GitHub:
git checkout main
git pull origin main
git branch -d feature/nueva-funcionalidad

Trunk-Based Development

# Trabajar directamente en main/trunk
# Commits pequeños y frecuentes
# Feature flags para funcionalidades no completadas
# CI/CD automatizado en cada commit

# Ejemplo con feature flag:
if (featureEnabled("nueva-funcionalidad")) {
    // nueva funcionalidad
} else {
    // comportamiento actual
}

Optimización y Mantenimiento

Limpieza de Repositorio

# Limpiar archivos no rastreados
git clean -fd              # Eliminar archivos y directorios
git clean -n               # Simular limpieza

# Recolectar basura
git gc                     # Garbage collection
git gc --aggressive        # Más agresivo

# Empaquetar referencias sueltas
git prune

# Eliminar referencias inalcanzables
git prune --expire now

# Ver tamaño del repositorio
git count-objects -vH
du -sh .git

Reescritura de Historial

# ¡ADVERTENCIA! No reescribir historial compartido

# Filtrar historial (remover archivos grandes)
git filter-branch --tree-filter 'rm -rf archivo_grande.zip' --prune-empty HEAD

# Reescribir autor de commits
git filter-branch --env-filter '
    if [ "$GIT_AUTHOR_NAME" = "Nombre Viejo" ]; then
        export GIT_AUTHOR_NAME="Nombre Nuevo"
        export GIT_AUTHOR_EMAIL="nuevo@email.com"
    fi
    if [ "$GIT_COMMITTER_NAME" = "Nombre Viejo" ]; then
        export GIT_COMMITTER_NAME="Nombre Nuevo"
        export GIT_COMMITTER_EMAIL="nuevo@email.com"
    fi
' --tag-name-filter cat -- --branches --tags

Aliases Útiles

# Configurar aliases avanzados
git config --global alias.tree "log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit --date=relative"

git config --global alias.unpushed "log --branches --not --remotes --oneline"

git config --global alias.amend "commit --amend --no-edit"

git config --global alias.count "!git rev-list --all --count"

git config --global alias.alias "!git config --get-regexp '^alias\.' | sed -e 's/^alias\.//' -e 's/ /\t=/'"

Best Practices

Commits

  • Hacer commits atómicos y pequeños
  • Mensajes descriptivos y consistentes
  • Usar presente imperativo: "Fix bug" no "Fixed bug"
  • Incluir issue numbers cuando corresponda

Ramas

  • Mantener main siempre estable
  • Usar nombres descriptivos para ramas
  • Eliminar ramas después de fusionar
  • Sincronizar regularmente con upstream

Colaboración

  • Hacer pull requests pequeños y enfocados
  • Incluir descripción clara de cambios
  • Responder a feedback rápidamente
  • Mantener conversación en el PR

Seguridad

  • Nunca incluir secrets en el repositorio
  • Usar SSH keys en lugar de HTTPS
  • Firmar commits importantes
  • Revisar permisos de archivos

Troubleshooting Común

Problemas Frecuentes

# Deshacer último commit (mantener cambios)
git reset --soft HEAD~1

# Deshacer último commit (descartar cambios)
git reset --hard HEAD~1

# Recuperar archivo eliminado
git checkout HEAD~1 -- archivo.txt

# Encontrar commit perdido
git reflog
git checkout HEAD@{2}

# Resolver "detached HEAD"
git checkout main
git branch temp-branch HEAD@{1}
git checkout main
git merge temp-branch
git branch -d temp-branch

# Limpiar cambios locales
git reset --hard HEAD
git clean -fd
git fetch --all
git reset --hard origin/main

Conclusiones

Dominar Git es un proceso continuo que va más allá de conocer comandos. La clave está en entender los conceptos subyacentes y aplicar los flujos de trabajo adecuados para cada situación.

Puntos Clave:

  • Practica comandos básicos hasta que sean automáticos
  • Elige un workflow y síguelo consistentemente
  • Haz commits pequeños y descriptivos
  • Usa herramientas de visualización para entender mejor
  • Mantén tu repositorio limpio y organizado
  • Aprende a resolver conflictos con confianza
  • Automatiza tareas repetitivas con hooks y aliases
