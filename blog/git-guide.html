Introducción

Git es la herramienta de control de versiones más utilizada en el desarrollo de software moderno. Dominar Git no solo significa conocer los comandos básicos, sino entender flujos de trabajo eficientes y técnicas avanzadas que mejorarán tu productividad y la calidad de tu código.

Esta guía está diseñada para desarrolladores que ya tienen conocimientos básicos de Git y quieren llevar sus habilidades al siguiente nivel. Aprenderás técnicas avanzadas, flujos de trabajo profesionales y mejores prácticas que te convertirán en un experto en control de versiones.

Fundamentos de Git

Antes de entrar en comandos avanzados, asegurémonos de entender los conceptos fundamentales que forman la base de Git:

Repository (Repositorio)

Un repositorio Git contiene todos los archivos de tu proyecto junto con el historial completo de cambios. Cada repositorio tiene una carpeta .git que almacena toda la información de versiones.

Working Directory, Staging Area, Repository

Git tiene tres áreas principales que debes entender:

  • Working Directory: Donde modificas tus archivos
  • Staging Area: Área donde preparas cambios para el siguiente commit
  • Repository: Donde Git guarda permanentemente tus commits

Commit

Un commit es una instantánea de tu proyecto en un momento específico. Cada commit tiene un ID único (hash SHA-1) y contiene los cambios, autor, fecha y mensaje descriptivo.

Branch (Rama)

Las ramas permiten trabajar en diferentes versiones del proyecto simultáneamente. La rama principal suele llamarse main o master.

Concepto Clave
Git es un sistema distribuido, lo que significa que cada desarrollador tiene una copia completa del historial del proyecto. Esto permite trabajar offline y mantiene redundancia de los datos.

Configuración Inicial

Una configuración adecuada es fundamental para un workflow eficiente. Configura tu entorno Git una vez y olvídate de problemas futuros.

Configurar Identidad

Configuración Básica
# Configurar nombre y email (obligatorio)
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

# Configurar editor por defecto
git config --global core.editor "code --wait"  # VS Code
git config --global core.editor "nano"         # Nano
git config --global core.editor "vim"          # Vim

# Configurar herramienta de diferencias
git config --global diff.tool "vscode"

# Configurar alias útiles
git config --global alias.st "status"
git config --global alias.co "checkout"
git config --global alias.br "branch"
git config --global alias.cm "commit -m"
git config --global alias.unstage "reset HEAD --"
git config --global alias.last "log -1 HEAD"

# Ver configuración actual
git config --list

Configuración de Seguridad

SSH Keys para GitHub
# Generar clave SSH Ed25519 (recomendado)
ssh-keygen -t ed25519 -C "tu@email.com"

# Iniciar agente SSH
eval "$(ssh-agent -s)"

# Añadir clave privada al agente
ssh-add ~/.ssh/id_ed25519

# Copiar clave pública al portapapeles
cat ~/.ssh/id_ed25519.pub | clip

# Probar conexión
ssh -T git@github.com
Seguridad
Usa siempre SSH keys en lugar de HTTPS para autenticación. Es más seguro y no requiere contraseña para cada operación.

Comandos Esenciales

Estos son los comandos que usarás diariamente. Master estos comandos y serás mucho más productivo.

Trabajo con Ramas

Gestión de Ramas
# Crear nueva rama y cambiar a ella
git checkout -b feature-nueva-funcionalidad

# Listar ramas locales
git branch

# Listar todas las ramas (incluyendo remotas)
git branch -a

# Cambiar a rama existente
git checkout develop

# Renombrar rama actual
git branch -m nuevo-nombre

# Eliminar rama local (solo si está mergeada)
git branch -d feature-completada

# Forzar eliminación de rama (con cambios no mergeados)
git branch -D feature-abandonada

Sincronización con Remoto

Trabajo Remoto
# Clonar repositorio
git clone git@github.com:usuario/repo.git

# Añadir remoto
git remote add origin git@github.com:usuario/repo.git

# Ver remotos configurados
git remote -v

# Enviar cambios al remoto
git push origin main

# Enviar nueva rama al remoto
git push -u origin feature-nueva

# Traer cambios del remoto
git pull origin main

# Traer cambios sin merge (solo fetch)
git fetch origin

# Sincronizar con rebase (más limpio)
git pull --rebase origin main

# Eliminar rama remota
git push origin --delete feature-antigua

Stashing

Guardado Temporal
# Guardar cambios temporalmente
git stash

# Guardar con mensaje descriptivo
git stash save "Trabajo en progreso de feature X"

# Listar todos los stashes
git stash list

# Aplicar último stash (y eliminarlo)
git stash pop

# Aplicar stash específico (sin eliminar)
git stash apply stash@{1}

# Ver contenido de un stash
git stash show -p stash@{0}

# Eliminar stash específico
git stash drop stash@{1}

# Eliminar todos los stashes
git stash clear

Flujos de Workflow

Elegir el workflow adecuado para tu equipo es crucial para mantener un desarrollo ordenado y eficiente.

Git Flow

Git Flow es un workflow extendido con múltiples ramas para diferentes propósitos:

  • main: Rama principal con código de producción
  • develop: Rama de desarrollo integrando todas las features
  • feature/*: Ramas para nuevas funcionalidades
  • release/*: Ramas para preparación de releases
  • hotfix/*: Ramas para fixes urgentes en producción
Git Flow Commands
# Iniciar feature desde develop
git flow feature start nueva-feature

# Finalizar feature y merge a develop
git flow feature finish nueva-feature

# Iniciar release desde develop
git flow release start v1.2.0

# Finalizar release y merge a main y develop
git flow release finish v1.2.0

# Iniciar hotfix desde main
git flow hotfix start fix-critico

# Finalizar hotfix y merge a main y develop
git flow hotfix finish fix-critico

GitHub Flow

Workflow más simple con solo dos ramas principales:

  • main: Siempre deployable y estable
  • feature branches: Todo el trabajo se hace en ramas de feature
  • Pull Requests: Para revisión y merge a main

GitLab Flow

Similar a GitHub Flow pero con ramas de ambiente:

  • main: Código de producción
  • environment branches: staging, production, etc.
  • feature branches: Desarrollo de nuevas funcionalidades
Recomendación
Para equipos pequeños o proyectos personales, GitHub Flow es generalmente suficiente. Para equipos grandes o proyectos críticos, considera Git Flow.

Técnicas Avanzadas

Estas técnicas te darán superpoderes con Git. Úsalas sabiamente.

Interactive Rebase

Rebase interactivo te permite reescribir el historial de commits de forma segura:

Rebase Interactivo
# Rebase interactivo de últimos 3 commits
git rebase -i HEAD~3

# Opciones disponibles:
# pick: mantener el commit como está
# reword: cambiar mensaje del commit
# edit: pausar rebase para modificar el commit
# squash: combinar con commit anterior
# fixup: combinar con commit anterior (descartando mensaje)
# drop: eliminar commit

# Ejemplo de reordenación y squash:
# pick 1234567 Primer commit
# squash 2345678 Segundo commit (se combina con el primero)
# pick 3456789 Tercer commit
# reword 4567890 Cuarto commit (cambiar mensaje)

Cherry-Pick

Aplicar Commits Específicos
# Aplicar commit específico en la rama actual
git cherry-pick abc1234

# Aplicar commit pero sin crear commit
git cherry-pick --no-commit abc1234

# Aplicar múltiples commits
git cherry-pick start..end

# Aplicar commit con edición del mensaje
git cherry-pick -e abc1234

Reset vs Revert vs Checkout

Diferencias Clave
# Reset: Mover HEAD y cambiar branch
git reset --hard HEAD~1  # Elimina cambios
git reset --soft HEAD~1   # Mantiene cambios en staging
git reset --mixed HEAD~1  # Cambios a working directory (default)

# Revert: Crear nuevo commit que revierte cambios
git revert abc1234

# Checkout: Cambiar branch o restaurar archivos
git checkout main
git checkout -- archivo.txt  # Restaurar archivo
git checkout abc1234 -- archivo.txt  # Restaurar desde commit

Bisect

Bisect es una herramienta poderosa para encontrar el commit que introdujo un bug:

Búsqueda Binaria
# Iniciar bisect para encontrar bug
git bisect start
git bisect bad  # Marcar commit actual como malo
git bisect good abc1234  # Marcar commit bueno

# Git te dará commits para probar
# Marca como bueno o malo hasta encontrar el culpable
git bisect good
git bisect bad

# Ver el estado actual
git bisect log

# Terminar bisect
git bisect reset

# Bisect automático con script
git bisect run ./test-script.sh

Resolución de Conflictos

Los conflictos son inevitables en desarrollo colaborativo. Aprende a resolverlos eficientemente.

Tipos de Conflictos

  • Same file, same lines: Ambas ramas modificaron las mismas líneas
  • File deletion: Una rama eliminó archivo que otra modificó
  • Renamed files: Conflictos en archivos renombrados
  • Binary files: Conflictos en archivos no textuales

Herramientas de Resolución

Comandos de Resolución
# Ver archivos con conflictos
git status

# Ver marcadores de conflicto
git diff

# Abrir herramienta de merge configurada
git mergetool

# Marcar conflicto como resuelto manualmente
git add archivo-conflictivo.txt

# Continuar merge/rebase después de resolver conflictos
git merge --continue
git rebase --continue

# Abortar merge/rebase si hay muchos conflictos
git merge --abort
git rebase --abort

# Ver nuestro lado vs su lado en conflictos
git checkout --ours archivo.txt
git checkout --theirs archivo.txt
Estrategia de Resolución
  1. Comunica con el otro desarrollador
  2. Entiende ambos cambios
  3. Combina manualmente si es necesario
  4. Prueba los cambios combinados
  5. Marca como resuelto y continúa

Optimización y Mantenimiento

Mantén tu repositorio Git limpio y optimizado para mejor rendimiento.

Garbage Collection

Limpieza del Repositorio
# Limpiar objetos no alcanzables
git gc

# Limpieza agresiva (más lenta pero más completa)
git gc --aggressive

# Limpiar archivos comprimidos inmediatamente
git gc --prune=now

# Ver estadísticas del repositorio
git count-objects -v

# Ver tamaño del repositorio
du -sh .git

Reflog

Reflog es tu red de seguridad. Te permite recuperar commits "perdidos":

Recuperación con Reflog
# Ver historial de referencias
git reflog

# Ver reflog de una rama específica
git reflog main

# Restaurar commit perdido
git checkout HEAD@{2}

# Crear rama desde estado perdido
git branch rescue-branch HEAD@{3}

# Limpiar reflog (cuidado!)
git reflog expire --expire=now --all
git gc --prune=now

Large Files

Git LFS para Archivos Grandes
# Instalar Git LFS
git lfs install

# Rastrear tipos de archivo grandes
git lfs track "*.psd"
git lfs track "*.zip"
git lfs track "*.mp4"
git lfs track "*.pdf"

# Añadir .gitattributes al repositorio
git add .gitattributes
git commit -m "Configure Git LFS"

# Mover archivo existente a LFS
git lfs migrate import --include="*.zip"

# Ver qué archivos están en LFS
git lfs ls-files

Puntos Clave

  • Git es más que comandos, es entender flujos de trabajo
  • Configura correctamente tu identidad y herramientas desde el inicio
  • Usa ramas para mantener el código principal siempre estable
  • Aprende a resolver conflictos de manera eficiente y colaborativa
  • Mantén tu repositorio limpio con garbage collection regular
  • Elige el workflow adecuado para tu equipo y síguelo consistentemente
  • Usa herramientas visuales cuando sea necesario, pero domina los comandos
  • Practica regularmente para mantener fluidez con los comandos básicos
  • Documenta tus convenciones de commits y workflows
  • Siempre prueba antes de hacer push a ramas principales

Próximos Pasos

Ahora que tienes una base sólida de Git avanzado, considera explorar:

  • Git Hooks: Automatiza tareas con hooks pre-commit, post-merge, etc.
  • Submodules: Gestiona dependencias de otros repositorios
  • Worktrees: Trabaja con múltiples branches simultáneamente
  • Git Attributes: Configura comportamiento específico por archivo
  • Filter Branch: Reescribe historial masivamente (con precaución)
Recomendación Final
La mejor manera de aprender Git es practicando. Crea un repositorio de prueba, experimenta con estos comandos, y no tengas miedo de cometer errores. Con reflog, siempre puedes recuperar cualquier cambio.
