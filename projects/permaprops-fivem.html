Permaprops-script-fivem

Script de Objetos Permanentes

Script para FiveM que permite la colocación de objetos permanentes. Compatible tanto con ESX como con QB frameworks, ofreciendo una solución completa para la gestión de objetos persistentes en servidores roleplay.

Ver en GitHub Volver a Proyectos

Detalles del Proyecto

Características Principales

  • Colocación de objetos permanentes en el mapa
  • Compatible con ESX y QB frameworks
  • Sistema de permisos basado en roles
  • Interfaz intuitiva para gestión de objetos
  • Soporte para categorías personalizadas
  • Sistema de backup y restauración

Tecnologías Utilizadas

Lua Scripting
FiveM
ESX Framework
QB Framework
MySQL
Map Editor

Funcionalidades Avanzadas

  • Editor visual de objetos en tiempo real
  • Sistema de categorización de objetos
  • Gestión de distancias de renderizado
  • Optimización de rendimiento para servidores grandes
  • Sistema de logging completo
  • Integración con sistemas de inventario

Seguridad y Rendimiento

  • Validación de permisos en servidor
  • Optimización de streaming de objetos
  • Límites configurables por usuario
  • Protección contra spam de objetos
  • Monitoreo de rendimiento en tiempo real
  • Sistema de cleanup automático
